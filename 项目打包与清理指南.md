# API 平台测试系统 - 打包与清理指南

本指南用于指导在项目进行重新打包（生成 Release）或上传至 GitHub 仓库时，如何对当前目录下的文件进行清理和整理。

---

## 1. 第一大类别：必须整理和上传到 GitHub 仓库的文件 (A 类)
这类文件是系统的核心代码和公开的开源说明文档，必须包含在代码仓库和交付源码包中：

*   **核心源码目录**：`api_probe_system/` （注：需排除内部忽略的敏感数据 `platforms.enc`）
*   **启动服务入口**：`run_web.py`
*   **自动化测试用例**：`tests/`
*   **开源必要文件**：`LICENSE`、`README.md`
*   **版本控制忽略规则**：`.gitignore`、`api_probe_system/config/.gitignore`
*   **报告模板规范**：`reports/报告模板规范.md` （仅保留此空模板，用于新用户生成报告时的格式参考）

---

## 2. 第二大类别：不上传到 GitHub 仓库的文件 (B / C 类)
这类文件在 `.gitignore` 规则中已被配置为忽略，绝不能提交至公共仓库。具体处理方式如下：

### (a) 必须在本地保留的 (绝对不能删除)
这类文件是运行和测试本地程序所必须的本地资产，物理删除会导致程序崩溃或本地配置丢失：
*   **本地可执行主程序**：`API Probe.exe` （双击运行系统的编译成品，必须保留在本地）
*   **本地加密平台数据库**：`api_probe_system/config/platforms.enc` （加密保存了您的私密 API 密钥，删除会导致配置清空，必须保留）
*   **用户偏好配置文件**：`api_probe_system/config/user_config.yaml` （保存了本地测试偏好，删除会导致设置重置，必须保留）

### (b) 可以保留的 (建议在本地归档/保留，但绝不上传)
这类文件是您的核心文档、推广运营方案、项目管理跟踪以及历史探测的报告资产。删除它们不会影响系统运行，但在本地具有极高的保留价值：
*   **私密 API 密钥文档**：`【重要】API平台列表.md` （记录了各大 API 平台的原始密钥，建议加密或安全存放于本地，绝不删除）
*   **实操指南与规格文档**：`Ollama云端模型拉取与部署指南.md`、`API平台测试系统需求规格说明书 v1.0.md`
*   **运营与推广文档**：`开源推广方案.md`、`手动推广操作指南.md`
*   **项目状态与需求文档**：`最新项目信息和任务状态.md`、`需求跟踪.md`、`项目具体情况分析报告.md`、`项目进度与交接.md`
*   **历史生成的探测报告**：`reports/` 目录下除模板外的所有 `.md` 和 `.json` 报告文件（注：**每个平台只保留最新的一份报告，旧的报告全部删除**，以保持报告目录整洁）

### (c) 应该物理删除的 (无用缓存与敏感草稿)
这类文件是开发过程中的临时缓存或带硬编码密钥的单步调试脚本。在打包或提交前**必须物理删除**，既可以显著减小打包体积，也能百分之百防范密钥泄露：
*   **Python 测试缓存**：`.pytest_cache/` （自动化测试产生的缓存，下次运行测试时会自动重新生成）
*   **IDE 运行缓存**：`.claude/` （IDE 历史临时上下文）
*   **本地历史版本备份**：`Version Control/`、`Version_Control/` （历史演进代码备份。注意：系统已在 Session 目录创建备份，主目录下的应安全删除，防敏感历史代码泄露）
*   **历史打包压缩包**：`api-probe-system_V1.0.zip` （旧版打包压缩文件）
*   **本地调试临时草稿**：`scratch/` （存放了开发过程中的调试脚本，其中部分脚本带有硬编码的临时密钥）
