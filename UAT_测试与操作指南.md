# API 平台智能探测系统 — UAT 操作与测试指南

本指南用于指导用户在系统交付后，对 **命令行探测引擎**、**FastAPI Web API 服务**、**实时打分计算与多平台横向对比报告生成器**以及**「深空探测站」React Web 前端管理界面**进行完整的 UAT（用户接受度测试）。

---

## 一、 测试前置条件准备

In 执行任何测试用例之前，请确保以下环境和密码已配置完毕：

1. **代理服务器可用性**：
   - 探测外部平台（如 BlazeAI, Groq）高度依赖局域网代理。请确保在运行测试的终端或后台中设置了 `HTTPS_PROXY` 环境变量：
     ```bash
     # Windows Cmd
     set HTTPS_PROXY=http://127.0.0.1:7890

     # Windows PowerShell / Git Bash
     export HTTPS_PROXY=http://127.0.0.1:7890
     ```

2. **OS Keychain 主密码解锁**：
   - 系统会自动尝试从 OS Keychain 中拉取名为 `api-probe-system` 且用户名为 `master-password` 的条目进行无感解锁。
   - 若您是首次在当前电脑运行，执行探测或运行 Web 服务时终端会交互式提示您输入主密码（解密平台注册表所需的 9 位密码），输入成功后主密码会自动同步并保存至 Windows 凭据管理器，后续测试即可实现全自动解锁。

## 二、 后端与命令行测试 (CLI & API)

本部分聚焦于命令行工具和后端 API 端点的功能与数据契约校验。

### Case 1: CLI 标准模式单平台与多平台探测测试
* **测试目的**：验证命令行主入口能够正常解锁注册表并运行标准 0~3 阶段测试，物理生成探测报告。
* **操作步骤**：
  1. 打开终端（PowerShell 或 Git Bash），进入到项目路径中的 `api_probe_system/` 目录：
     ```bash
     cd "d:/同步盘/BaiduSyncdisk/Antigravity/ProjectAsset/API平台测试 260601/api_probe_system"
     ```
  2. 执行对 `Groq` 平台的标准探测命令：
     ```bash
     python run.py --platforms Groq --mode standard
     ```
  3. 也可以执行交互菜单挑选平台：
     ```bash
     python run.py
     # 终端将打印 1~18 个可用平台的编号，输入对应数字（如 1）按回车即可。
     ```
* **预期结果**：
  1. 控制台正确打印出：`[OK] 已从 OS Keychain 自动解锁加密配置`；
  2. 打印出平台基本信息（base_url, api_key 脱敏后缀, 响应阈值参数）；
  3. 最终显示：`[OK] Groq 耗时 XX.Xs`，并打印出生成的 Markdown 报告物理路径；
  4. 检查 [reports/](file:///d:/同步盘/BaiduSyncdisk/Antigravity/ProjectAsset/API平台测试%20260601/reports/) 目录，新生成了 `Groq_探测报告_yymmdd_hhmm.md` 文件。

### Case 2: CLI 深度模式 (Deep) 边界与稳定性探测测试
* **测试目的**：验证 Stage 4（最大上下文与 RPM 探测）及 Stage 5（多轮重复调用稳定性分析）可被完整编排并输出数据。
* **操作步骤**：
  1. 在 `api_probe_system/` 目录下运行针对 `Groq` 平台的 `deep` 模式探测：

* **预期结果**：

  2. 探测完成后，打开生成的 `reports/Groq_探测报告_*.md` 报告：
     - **「第七部分：边界探测」**表格正确包含了模型测出的“最大输出 token”、“最大上下文长度”、“速率限制(RPM)”以及对应的「速率获取来源」标志；
     - **「第八部分：稳定性分析」**表格中正确展示了前 3 个可用模型的 5 次调用“成功率”、“平均延迟”以及根据延迟和异常分类得到的“星级推荐”与错误模式。

### Case 3: Web 平台配置 API 读取与脱敏校验
* **测试目的**：验证 FastAPI 后端在返回平台列表或详情响应时，能动态解密注册表，安全脱敏展示，并提取物理报告进行打分和推荐模型计算。
* **操作步骤**：
  1. 在项目根目录下，启动 Web 引导程序：
     ```bash
     cd "d:/同步盘/BaiduSyncdisk/Antigravity/ProjectAsset/API平台测试 260601"
     python run_web.py
     ```
     *(Uvicorn 后端将在 `http://127.0.0.1:8080` 启动运行)*
  2. 在终端中向平台列表接口发起 HTTP GET 请求：
     - **PowerShell (方式 A，推荐)**:
       ```powershell
       Invoke-RestMethod -Uri "http://127.0.0.1:8080/api/platforms"
       ```
     - **Windows PowerShell 中若使用 curl (方式 B)**:
       由于 PowerShell 的 `curl` 别名污染限制，如需使用 curl 请显式指定并运行 `curl.exe` 后缀（避免调用原生 Invoke-WebRequest 报不支持参数 -s 的错误）：
       ```powershell
       curl.exe -s http://127.0.0.1:8080/api/platforms
       ```
     - **Git Bash / Linux (方式 C)**:
       ```bash
       curl -s http://127.0.0.1:8080/api/platforms
       ```
* **预期结果**：
  1. 返回全量 18 个平台的 JSON 数据列表，结构完整；
  2. 校验 JSON 数组中的安全及打分字段：
     - `"api_key"` 字段格式必须已被脱敏（如 `"gsk_...0xOk"`）；
     - 已存在 `.md` 报告的平台（如 `Groq`, `BlazeAI`），其 `"score"` 字段为非空的浮点数值（如 `85.4`），未测试平台显示为 `null`；
     - `"top_models"` 字段已自动填充该平台评分最高的前 3 个可用模型 ID。

### Case 4: Web 异步测试调度与 SSE 事件推送校验
* **测试目的**：验证后台测试任务支持异步触发派发，并通过 SSE（Server-Sent Events）连接实时广播进度包。
* **操作步骤**：
  1. **建立事件订阅**：
     打开一个窗口，发起对事件流接口的监听。根据您的终端选择以下方式之一：
     
     * **方式 A：使用浏览器直接打开（最直观推荐）**
       直接在浏览器中打开链接：👉 [http://127.0.0.1:8080/api/tests/events](http://127.0.0.1:8080/api/tests/events)
       *(浏览器将维持长连接并实时在页面展示推送回来的文本流)*
       
     * **方式 B：在 Windows PowerShell 中运行**
       注意：PowerShell 默认将 `curl` 设为 `Invoke-WebRequest` 的别名，直接运行 `curl -N` 会报错。您必须显式调用真实的 `curl.exe` 后缀：
       ```powershell
       curl.exe -N http://127.0.0.1:8080/api/tests/events
       ```
       
     * **方式 C：在 Git Bash / Linux / WSL 中运行**
       ```bash
       curl -N http://127.0.0.1:8080/api/tests/events
       ```
  2. **异步触发测试**：
     在另一个终端 A 中，向测试触发接口发送 `POST` 请求（指定标准探测平台 Groq）：
      - **PowerShell (推荐方式，可直接复制)**:
        ```powershell
        Invoke-RestMethod -Uri "http://127.0.0.1:8080/api/tests/start" -Method Post -ContentType "application/json; charset=utf-8" -Body '{"platforms": ["Groq"], "mode": "standard"}'
        ```
      - **Git Bash / Linux (使用 curl)**:
        *(注意：由于 PowerShell 别名冲突以及单双引号转义剥离机制，Windows PowerShell 环境下极易执行 curl POST 报错，请仅在 Git Bash 或 Linux 中使用以下命令)*
        ```bash
        curl -X POST http://127.0.0.1:8080/api/tests/start -H "Content-Type: application/json" -d '{"platforms": ["Groq"], "mode": "standard"}'
        ```
* **预期结果**：
  1. 终端 A 立即返回类似 `{"message": "成功拉起 1 个平台的异步探测任务", "task_id": "..."}` 的 JSON 包，不发生阻塞；
  2. 终端 B 随着测试的进行，开始源源不断接收到流式推送。流数据带 `data:` 前缀，格式为 JSON：
     - `{"platform": "Groq", "stage": "准备中", "status": "started", "message": "..."}`
     - 各阶段事件（如 `"stage": "连通性检测"`, `"status": "running"` / `"success"` 等）；
     - 测试结束时输出最终的成功包，并附带最终的物理报告名和计算出的得分。

### Case 6: 发现处理器故障与平台兼容性告警决策中心测试
* **测试目的**：验证当平台在探测时因为非标接口而发现失败，系统能自动将错误汇总至告警文件，且在前端右上角通过“脉冲呼吸灯”和“告警抽屉”引起注意，并在卡片上显眼标注，给出开发新处理器的决策提示。
* **操作步骤**：
  1. 向平台列表中添加或修改一个错误的测试平台（例如 Base URL 故意填写一个不存在的地址 `http://127.0.0.1:9999`），并触发该平台的标准探测；
  2. 探测报错后，观察主页右上角；
  3. 点击右上角的 ⚠️ 告警中心按钮；
  4. 检查错误平台卡片标题右侧及卡片外观。
* **预期结果**：
  1. 在**没有平台报错的正常状态下**，右上角的警告中心按钮展示为一个**淡灰色的 Shield (安全盾牌) 图标**，没有红色发光与任何脉冲呼吸灯动效，保持静默；
  2. 当**有平台探测发生故障报错后**，该按钮图标会立即自动变更为**红色的 ⚠️ (AlertTriangle) 图标**，且四周产生向外扩散的红色脉冲扩散波纹（呼吸灯效果），并附带红色数字徽标显示故障平台数 `1`；
  3. 点击该图标后，从右侧滑出一个玻璃磨砂的 **告警抽屉**，里面列出该平台在 `openai` 处理器下发生的故障及时间，并醒目提示：*“检测到非标准 OpenAI 格式。建议为此平台开发并注册新的 ModelDiscoveryHandler”*；
  3. 主页面上，该平台卡片的右上角出现闪烁的 **“⚠️ 兼容性异常”** 红色 Badge，Hover 悬停时能气泡化展示具体失败原因，且卡片边缘发出红色的警告呼吸光圈；
  4. 当用户在配置弹窗中修正该平台的 Base URL 或密钥并成功运行探测后，右上角的警告 Badge 自动消退，且卡片警示红标与发光圈自动清除。

## 四、 预期报告校验标准

当您物理打开单平台 Markdown 报告（如 `Groq_探测报告_*.md`）或在网页渲染时，请核对是否符合以下设计规范：

1. **执行摘要**：
   - 包含健康度（良好🟢、差🔴、中等🟡、异常🔴）与模型可用率；
   - 包含“一句话结论”，指出平台基本连通状况、模型可用占比以及不可用模型的主要原因分类。
2. **能力支持率**：
   - 「四、可用模型能力矩阵」的下方，应伴有流式、非流式、工具调用、视觉等 8 大能力支持率的纯文本字符进度条（如 `流式 ████████░░░░ 80%`）。
3. **问题分类可解释性**：
   - 「五、可用模型问题统计」和「六、不可用模型分析」应明确展示失败的“问题成因”（如 *账户无权访问*、*后端格式不规范*、*聚合站后端 provider 不可达* 等），拒绝以“未知 (UNKNOWN)”笼统代替。

## 五、 异常排查与 Keychain 密钥维护

如果您在 UAT 期间遇到任何异常，请按以下步骤自查：

1. **报告显示连通性可达，但可用率极低，且大量报错 HTTP 403**：
   - 这是正常的平台限制行为。代表您的该平台 API Key 账户由于额度欠费或账户等级原因，无法调用大部分模型。可以在报告第六章“不可用模型分析”的“错误摘要”列看到具体的拒接信息。
2. **需要更换或重置主密码**：
   - 如果需要手动修改 Keychain 里的主密码（例如密码遗忘或需要清空）：
     - 打开 Windows 搜索，输入 `凭据管理器` (Credential Manager) 并打开；
     - 选择 `Windows 凭据`；
     - 找到普通凭据中的 `api-probe-system` 这一项；
     - 点击展开，选择 `编辑`（修改主密码明文值）或 `删除`（删除后下次运行 run_web.py 会重新提示输入）。
